# IMvision

IMvision is one of the API-security vendors evaluated head-to-head against [Noname Security](/notes/noname-security) in Kyle's 2021 Noname diligence. Across multiple expert calls it gets booted quickly for a specific reason: it is "very focused on giant companies with no cloud presence" and "[doesn't] support AWS, Azure, etc. — only focused on on-prem environments" and "old-school companies." One CISO ([Rapyd](/notes/rapyd)) and [David Dean](/notes/david-dean) of DGDean both list it among platforms they evaluated (alongside [Salt Security](/notes/salt-security) and [NeoSec](/notes/neosec)) before choosing Noname. Dean adds the lineage detail that "IMvision started as an API Gateway but expanded into security," and that he "didn't want to have security rely on the gateway because holes could unknowingly open up." In the diligence it reads as the legacy/on-prem-bound option in a category that experts believed was moving to cloud-native, traffic-mirroring approaches.

**Context:** Imvision is an Israel-based API security company focused on API threat detection and anomaly analysis; in this era it was positioned toward large on-premise enterprises.
