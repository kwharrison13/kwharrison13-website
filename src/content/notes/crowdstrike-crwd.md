# CrowdStrike ($CRWD)

**One-line definition:** *Leading endpoint security platform (EDR/MDR); used as the analogy for what [Salt Security](/notes/salt-security) could become in API security — "CrowdStrike to Kong's ServiceNow."*

## How sources describe it

- Central analogy in the Salt Security diligence: [ServiceNow ($NOW)](/notes/servicenow-now) → CrowdStrike :: [Kong](/notes/kong) → [Salt Security](/notes/salt-security) — i.e., the API management layer spawns a dedicated security company on top. ([Atanu Dasgupta](/notes/atanu-dasgupta), Salt Security Diligence — 2021)
- "You have to have endpoint security (CrowdStrike, SentinelOne). You have to have a Firewall (Palo Alto, CheckPoint). You need a CSPM (Orca, Wiz.io). You need to have an API security tool." — the "must-have" stack framing. ([Nir Rothenberg](/notes/nir-rothenberg), Salt Security Diligence — 2021)
- Note: CrowdStrike doesn't get into Application Security because "it's very different than endpoint security" — limits for gateway-based expansion. ([Atanu Dasgupta](/notes/atanu-dasgupta), Salt Security Diligence — 2021)
- [NeoSec](/notes/neosec) was described as feeling "geared towards being acquired by CrowdStrike, like 'CrowdStrike for API Security.'" ([Nir Rothenberg](/notes/nir-rothenberg), Salt Security Diligence — 2021)
