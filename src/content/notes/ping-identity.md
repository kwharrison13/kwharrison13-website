# Ping Identity

**One-line definition:** *Identity and access management platform; acquired Elastic Beam (API security) but API wasn't a core focus because "API Gateway buyers aren't the security teams."*

## Where it shows up

- Salt Security Diligence — 2021 — [Michael Isbitski](/notes/michael-isbitski) mentioned Ping Identity's acquisition of Elastic Beam; API security wasn't prioritized because the buyer persona (API gateway teams) isn't security-focused.
- Identity & Access Management Diligence — 2021 — referenced in IAM market context.

## Related concepts

- [API Security](/notes/api-security) — Ping's Elastic Beam acquisition was an API security play, but de-prioritized because API gateway buyers aren't security teams.
- [Salt Security](/notes/salt-security) — the API security company under diligence against which Ping/Elastic Beam was discussed as a competing approach.
- [Michael Isbitski](/notes/michael-isbitski) — analyst who cited Ping's Elastic Beam acquisition and the buyer-persona mismatch.
