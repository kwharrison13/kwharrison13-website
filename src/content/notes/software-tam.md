# Software TAM

Kyle's sibling page to [TAM](/notes/tam), tracking how big the market for software *itself* can become as the ability to create software expands. The recurring framing comes from the Replit reading: today "fewer than 30 million people in the world know how to code," and most of us "still can't build" the software we use — which Kyle tags #[Software TAM](/notes/software-tam). [Replit](/notes/replit)'s wager (via [Amjad Masad](/notes/amjad-masad)) is that the ability to write code should become "as ubiquitous as the ability to write essays," giving a billion people the chance to build online. The TAM grows not by capturing existing developers but by minting new ones, and by better serving "the millions of small niches that make up the internet" (Niche Markets).

This pairs with the parent [TAM](/notes/tam) page's core idea — that a market's size is a function of imagination and execution, not a fixed ceiling ("Sometimes there are small markets. And sometimes there are small minds"). The page was originally seeded from a [Quick Capture](/notes/quick-capture) note pointing to a Wikipedia [list of the largest software companies](https://en.wikipedia.org/wiki/List_of_the_largest_software_companies) as a reference for sizing software markets.

**Context:** "Software TAM" is the venture/strategy question of how large the total revenue opportunity for software can grow — a number that keeps expanding as low-code/no-code, AI code generation, and developer tooling lower the barrier to building, enlarging the population of potential software creators.
