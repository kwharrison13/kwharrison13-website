# Developer Productivity

An investment-thesis lens recurring in Kyle's diligence notes, where developer productivity is treated as a structural tailwind for infrastructure and security startups. In Coder - Diligence, it is one of three drivers behind the high-level thesis that "development will move to the cloud, the same way that Microsoft Word moved to Google Docs" — alongside a shift-left in [Application Security](/notes/application-security) and the move to [Remote Work](/notes/remote-work) — with cloud development environments cast as the productivity-enhancing substrate.

In the September 17th, 2021 — Wiz.io Pre-Brief (Shardul Shah) note, developer productivity is framed as the *tension* security tooling must resolve. Because individual developers can't fully reason about the security implications of every cloud configuration, security leaders are stuck choosing between tightening processes (slowing dev velocity) and granting flexibility (bearing the security consequences) — a bind that produces incidents like [CapitalOne](/notes/capitalone)'s open S3 bucket. Wiz's pitch is to dissolve that trade-off by giving teams an up-to-the-minute view of cloud risk so they can move fast *without* sacrificing developer productivity.

**Context:** "Developer productivity" (often "developer experience" / DevEx) is a software-industry concern with how tooling, environments, and process friction affect how quickly and safely engineers ship code — a frequent thesis area for dev-tools and cloud-security investors.

## Where this appears

- Coder - Diligence — one of three drivers (with application-security shift-left and remote work) behind the "development moves to the cloud" thesis
- September 17th, 2021 — Wiz.io Pre-Brief (Shardul Shah) — the velocity-vs-security trade-off Wiz aims to dissolve with real-time cloud risk visibility
