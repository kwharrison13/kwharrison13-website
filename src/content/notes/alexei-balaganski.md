# Alexei Balaganski

**One-line:** *Analyst at [KuppingerCole](/notes/kuppingercole) specializing in identity and cybersecurity; has covered the API security space since 2014; argues for data-at-point-of-creation protection over component-level security.*

## What we know

- Analyst at [KuppingerCole](/notes/kuppingercole), a European analyst firm of ~50 people specializing in identity and cybersecurity. Also does market research, events, and advisory services.
- LinkedIn: https://www.linkedin.com/in/balaganski/
- Has covered API security since 2014; by 2021 counted 31 meaningful vendors (excluding WAFs).
- Key thesis: "Why would you want to protect individual components of your infrastructure when you can just protect your data at the point of creation?" — close relationship between API security and data access controls. (Source: Salt Security Diligence — 2021)
- View: network security vendors ("dinosaurs") are fighting for relevance; data protection is where the whole cybersecurity market is moving.
- Author of: API Management and Security (KuppingerCole Leadership Compass); API Management & Security Buyer's Compass.

- Salt Security Diligence — 2021 — analyst session, October 12, 2021, alongside [Martin Kuppinger](/notes/martin-kuppinger).
