# Jenkins

Jenkins is an open-source automation server for [CI/CD](/notes/ci-cd) pipelines that appears in Kyle's corpus as the legacy incumbent of the continuous-integration ecosystem. In the Noname Security - Diligence notes, an expert lists Continuous Integration along with [Jira](/notes/jira), [Confluence](/notes/confluence), and "even Jenkins" as part of a developer-tooling stack that is "all ripe for disruption" — tools that "do anything you need them to do at pretty significant cost." On the [CI/CD](/notes/ci-cd) concept page, Jenkins is cast in the same role: the legacy open-source automation server representing the older CI/CD ecosystem, against which newer entrants and security vendors (e.g. shift-left, [DevSecOps](/notes/devsecops) tooling) position themselves.

**Context:** Jenkins is a widely-used open-source automation server, originally forked from Hudson, for building, testing, and deploying software via configurable CI/CD pipelines. It is known for its large plugin ecosystem and self-hosted flexibility, and is often treated as the legacy reference point against newer pipeline tools like GitHub Actions, GitLab CI, and CircleCI.
