# API Economy

In Kyle's Pave Diligence — 2021 dossier, the "API Economy" names the investment thesis that Pave belongs to: companies that win by aggregating a category of previously-siloed data into a real-time, integration-driven system of record. The dossier maps the pattern explicitly — financial data → [Plaid](/notes/plaid), communications data → [Twilio ($TWLO)](/notes/twilio), payment data → [Stripe](/notes/stripe), and **compensation data → Pave**. The shared logic is a give-to-get data network: Pave offers free benchmarking in exchange for participation, building a proprietary dataset (1,500+ contributing companies, 70+ VC funds) whose value compounds with every new contributor — "an API-driven real-time data tool that adds more visibility to this process than ever before."

The concept also surfaces in [API](/notes/api), where the [Salt Security](/notes/salt-security) diligence transcript frames APIs as a large but under-secured integration surface — the flip side of the same trend. Where the API Economy thesis celebrates APIs as the connective tissue that lets a Pave or a Plaid pull data across HRIS, ATS, and cap-table systems in real time, the API-security angle notes that this same proliferating surface is exposed to brute-forcing, unauthenticated access, and scraping.

**Context:** "API economy" is a common venture/industry term for the business models built on application programming interfaces as products — companies like Stripe, Twilio, and Plaid that monetize programmatic access to a service or dataset, and the broader shift toward composable, integration-first software.
