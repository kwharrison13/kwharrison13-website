# Information Security

In Kyle's [Wiz.io](/notes/wiz-io) pre-brief from [Shardul Shah](/notes/shardul-shah), information security shows up as the market category Wiz is attacking — specifically cloud infrastructure security. The thesis: cloud platforms (e.g. AWS) are easy to adopt but introduce significant complexity to infrastructure management and security, fragmenting customers across many easy-to-spin-up resources with "no single source of truth" and a deluge of signals requiring manual triage. Wiz builds a full-stack, agentless, context-rich product to identify and manage that risk — positioned to do for Security what [Datadog ($DDOG)](/notes/datadog) did for Observability.

The note frames the core tension of modern information security as a tradeoff against [Developer Productivity](/notes/developer-productivity): security leadership is stuck "either tighten up security processes and slow down dev velocity, or allow developers flexibility and bear the potential security consequences," compounded by multiple teams and multiple clouds, and exemplified by failures like CapitalOne's open S3 bucket. Wiz's pitch is to resolve that tradeoff with an up-to-the-minute view of security across all cloud resources, bringing context to bear so security signals are prioritized and actionable.

**Context:** Information security (infosec) is the practice of protecting data and systems from unauthorized access, disclosure, modification, or destruction. Cloud security — the subdomain Wiz targets — addresses the distinct challenges of securing dynamic, ephemeral cloud infrastructure rather than static on-premise systems.
