# Low Code / No Code

The market category Kyle tracks through his 2021–2022 dev-tools diligence — software that lets non-engineers (and engineers on drudge work) build apps, internal tools, and workflows with little or no hand-written code. Kyle thinks in terms of a "Low Code / No Code market map" with a *spectrum* of technical demand: [Retool](/notes/retool) sits across that spectrum and still "requires some SQL or JavaScript" rather than being fully no-code, serving both self-upskilling BizOps teams and technical teams automating drudgery ([Julie Chen](/notes/julie-chen) of Retool framed it this way in their June 2022 call). A recurring thesis in his notes is that the durable opportunity is the *professionalization* of the category: as he recorded reading up on Airplane.dev, "so many low-code tools are effectively platformed hacking. The ability to have SSO, permissions, self-hosted, etc. changes the narrative" — turning "duct-taped scripts and cron jobs into safe, reusable tools."

The theme also frames Kyle's read on the category's edges. In his [Replit](/notes/replit) notes he flags "the no-code movement" as one competitive vector — "maybe being able to string together no-code tools will replace the need to code for most people (although that starts to look like coding itself)." On the education side, Ben Tossell of Makerpad (acquired by [Zapier](/notes/zapier)) was "really focused on improving the educational experience for getting into Low Code / No Code" and on being a "No Code Founder." And the category extends into AI: Actively, a diligence subject, is positioned as a "no-code AI platform for hypothesis-driven subject-matter experts" ("Tableau for AI" + "Retool for AI"), distinguished from data-science GUIs like DataRobot and Dataiku that are nominally "no-code" but still demand DS knowledge.

**Context:** "Low-code / no-code" refers to development platforms that replace or minimize hand-written code with visual builders, prebuilt components, and configuration — spanning internal-tools builders, workflow automation, and app builders aimed at "citizen developers." The pitch is faster delivery and broader access to building; the recurring critique (echoed in Kyle's notes) is that as use cases get complex, the abstraction "starts to look like coding itself."
