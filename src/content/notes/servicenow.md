# ServiceNow

**One-line definition:** *Enterprise workflow / IT service management platform; used as the "$15B endpoint management analogy" in the Salt Security thesis — the management layer that spawned CrowdStrike.*

## How sources describe it

- Central analogy in the Salt Security diligence: ServiceNow → [CrowdStrike ($CRWD)](/notes/crowdstrike) :: [Kong](/notes/kong) → [Salt Security](/notes/salt-security) — the API management layer (Kong) spawns a dedicated security company (Salt). (Salt Security Diligence — 2021)
- "ServiceNow don't really do security. CrowdStrike is all over the place (EDR, MDR, etc.)." — management platform and security platform are different enough to justify separate companies. ([Atanu Dasgupta](/notes/atanu-dasgupta), Salt Security Diligence — 2021)
- Endpoints ($15B market) = ServiceNow → CrowdStrike; API Management ($22B market) = Kong → Salt Security. (Salt Security Diligence — 2021)

## Related concepts

- [CrowdStrike ($CRWD)](/notes/crowdstrike) — the security company that grew out of the endpoint-management layer; the "$15B endpoint" half of the analogy where ServiceNow is the management platform.
- [Kong](/notes/kong) — the API-management counterpart to ServiceNow in the thesis (Kong → Salt mirrors ServiceNow → CrowdStrike).
- [Salt Security](/notes/salt-security) — the API-security company occupying the same structural slot Salt's thesis assigns CrowdStrike.
- Salt Security Diligence — 2021 — the diligence where ServiceNow appears as the core market-sizing analogy.
- [Atanu Dasgupta](/notes/atanu-dasgupta) — source of the note that ServiceNow "don't really do security," distinguishing management platforms from security platforms.
