# Memorization

In [How To Take Smart Notes](/books/how-to-take-smart-notes), "Memorization" is the section where Sönke Ahrens reframes memory away from raw storage and toward connection and retrieval — the chapter Kyle anchors to the slip-box's real value. The argument: short-term memory is sharply limited (the classic "seven plus/minus two," possibly as few as four), so we need to delegate thoughts to an external system rather than waste working memory holding them. What memory artists actually do is *bundle* items meaningfully; and "it is so much easier to remember things we understand than things we don't," because understood things are connected through rules, theories, narratives, mental models, and explanations — which is exactly what the slip-box builds. The book quotes [Charlie Munger](/notes/charlie-munger)'s "latticework of theory" passage (from [Poor Charlie's Almanack](/books/poor-charlie-s-almanack)) here, tying memorization to a [Latticework](/notes/latticework) of mental models.

A second move is the reframing of forgetting as *active inhibition* — not the loss of a memory but a productive filter that keeps us from being flooded (the Shereshevsky example). The lesson is to shift attention from storage strength to retrieval strength: learning is about building connections and the right "cues" so the useful memory triggers when needed, which connects to [Spaced Repetition](/notes/spaced-repetition) (retrieval over re-reading) and [elaboration](/notes/elaborate) as the core mechanism. Kyle's notes on the section link it to [Elder Klebingat](/notes/elder-klebingat)'s "mind palace" system — attaching relevant information to a framework so it's easier to trigger — and to selectively forgetting in order to get "the gist" of an idea.
